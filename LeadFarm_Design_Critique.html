Overall Impression

LeadFarm has a striking, premium visual identity. The dark agricultural theme with glass-morphism cards, amber accents, and the photographic farm backdrop immediately communicates the domain. It feels like a high-end SaaS tool designed specifically for Algerian precision agriculture.

The biggest opportunity is polish at the edges: empty states are bare, contrast ratios fall short in several places, the background image competes with data when tables are populated, and the information architecture could guide new users better with progressive onboarding.

First Impression (2-Second Test)

What draws the eye first: The satellite map on the dashboard dominates the viewport, which is correct for a geospatial agriculture tool. Good choice.

Emotional reaction: Professional, modern, niche. The dark green + amber palette feels agricultural without being corny. The farm backdrop adds personality.

Is the purpose clear? Partially. The "Precision Agriculture Platform" subtitle is small and monospaced. A first-time user might not immediately understand what actions to take. The KPI cards all show "0" which gives no signal about what this tool tracks.

Visual Hierarchy

Dashboard

Eye flow: Title → KPI cards → Map → Sidebar. The map is the correct focal point for an agriculture tool. However, the five KPI cards (Produits, Traitements, Parcelles, Stock Bas, Alertes) all show "0" and have equal visual weight, making them feel like placeholder widgets rather than actionable metrics.

Improvement: Give the most critical KPI (Stock Bas or Alertes) a distinct accent treatment — red glow for critical alerts, for instance. Make the zero-state meaningful: "0 alertes" could show a green check instead of a warning triangle.

Stock Page

Eye flow: Title → Action buttons (Inventaire, Exporter, Sortie, Nouvelle Entrée) → Tabs → Filters → Empty table. The hierarchy is well-structured with the primary CTA ("Nouvelle Entrée") in amber at top-right.

Issue: Five KPI cards + five tabs + four filter buttons + category dropdown = too many navigation layers before content. Consider collapsing the filter pills into the tab content area.

Login Page

Eye flow: Logo icon → "LeadFarm" title → Form card → Submit button. Clean and focused. The golden CTA button is the right anchor point. The footer "LeadFarm v1.0" adds a nice professional touch.

Consistency

ElementIssueRecommendation
KPI Card Layout Dashboard KPIs use icon + number + label. Stock page uses the same pattern. Products page uses the same. Good consistency here, but the "54 PARCELLES" card on the dashboard shows a different number style (the "54" doesn't match the "0" pattern of other cards). Standardize: all KPI values should use the same font-size and weight across pages.
Button Styles Three button variants are used inconsistently: amber filled ("Nouvelle Entrée"), amber outlined ("Sortie"), and glass ("Inventaire", "Exporter"). The hierarchy is unclear—are outlined buttons secondary or tertiary? Establish a clear 3-tier system: Primary (amber filled), Secondary (glass with amber border), Tertiary (glass transparent). Document in a style guide.
Page Headers Each page has a green icon + title + subtitle card, which is great for wayfinding. But the icon styles vary: some use gradients, others are flat. The parcelles page header has a different padding from the stock page. Extract a shared <PageHeader> component with identical padding, icon treatment, and CTA placement.
Tab Navigation Stock page uses pill-style tabs. Other pages don't have tabs at all. The active tab style (amber background) is visually strong but the inactive tabs blend into the background too much. Give inactive tabs a subtle hover state and a slightly brighter text color (white/50 → white/60).
Typography Section labels use mixed case: "PRINCIPAL" / "STOCK" / "TERRAIN" / "GESTION" in the sidebar are uppercase, but table headers mix uppercase ("PRODUIT", "CATÉGORIE") with sentence case subtitles. The font-size for labels (10px) is at the edge of readability. Pick one approach: all-caps for labels at 11px with letter-spacing: 0.05em, or sentence case at 12px. Apply uniformly.

Accessibility

CheckStatusDetail
Color Contrast (body text) Fails white/85 on dark green ≈ 4.2:1 ratio. WCAG AA requires 4.5:1 for normal text. Bump to white/90 or increase text size to 16px+ (where 3:1 suffices for large text).
Color Contrast (muted text) Fails white/40 and white/45 used extensively for labels and subtitles. These are well below 3:1. Some labels like "Tlemcen, SBA" and "Precision Agriculture" are nearly invisible. Minimum: white/60 for any readable text.
Color Contrast (sidebar labels) Fails "PRINCIPAL", "STOCK", "TERRAIN", "GESTION" group labels at white/25 are purely decorative-level contrast. If they serve a navigation purpose, they need to be at least white/55.
Touch Targets Partial Sidebar links at py-2.5 (~40px height) are adequate. Map control buttons (+/-) and icon-only buttons (Layers, Target, Maximize) are small (~32px). WCAG recommends 44x44px minimum.
ARIA Roles Good Modals have role="dialog" and aria-modal. Dropdowns have role="listbox". Close buttons have aria-label. Error alerts use role="alert". Good foundation.
Keyboard Navigation Partial Tab focus is visible on form inputs but focus rings on glass-card buttons and sidebar links are hard to see against the dark background. Add a visible focus-visible ring in amber.
Login: Show/Hide Password Good Has aria-label that changes between "Masquer" and "Afficher". Well implemented.

What Works Well

  • Unique visual identity. The dark agricultural theme with glass-morphism is distinctive. It doesn't look like a generic admin template. The amber + green palette is cohesive and domain-appropriate.
  • Satellite map integration. Using real ArcGIS World Imagery for Tlemcen parcelles is a powerful feature. The legend (treated, untreated, in-progress) is well-positioned.
  • Movement modal form design. The four-type selector (Entrée/Sortie/Retour/Transfert) with color-coded active state is intuitive. The traceability section (Fournisseur, BL, Lot) is professional.
  • Sidebar information density. Domain indicator ("Domaine Khelifa"), live/local status, and low-stock badge are all useful at-a-glance signals without cluttering the navigation.
  • Login page atmosphere. The full-bleed farm backdrop with the centered glass card is visually memorable. It sets the tone before the user even logs in.
  • French localization. Fully French UI with proper agricultural terminology (phytosanitaire, parcelles, traitements) shows domain expertise.
  • API hardening. Auth on every route, Zod validation, RLS policies — the backend is properly secured for production.

Priority Recommendations

  1. Tame the background image on data pages.
    The farm backdrop is the single biggest readability issue. On the stock, products, and suppliers pages, the tractor/orchard imagery directly underlays data tables. Add a near-opaque overlay (bg-[#0f1a0f]/90) for the main content column, or switch to a subtle CSS gradient for inner pages while keeping the photo on login only. This one change will dramatically improve scanability.
  2. Design meaningful empty states.
    Every data page (stock, products, suppliers, treatments) currently shows "0" or "Aucun X trouvé" with no guidance. Create illustrated empty states with: a relevant SVG illustration, a message explaining the benefit of adding data, and a single clear CTA button. First-run experience is make-or-break for adoption.
  3. Fix contrast ratios.
    Audit every text color. Set a floor of white/60 for any text that conveys meaning (currently many labels use white/25-white/40). This is both an accessibility requirement and a readability improvement for users working in bright environments (which farmers frequently are).
  4. Simplify the movement modal.
    Show one quantity input, not four. The type selector already communicates direction. Remove the disabled inputs and replace with a single "Quantité (kg/L)" field. Add inline validation and a confirmation preview before submission.
  5. Add responsive mobile handling.
    The sidebar CSS has max-lg:-translate-x-full for mobile, but the hamburger menu trigger needs to be verified at real device widths. Test on iPhone SE (375px) and iPad (768px). The KPI card row should stack to 2x2 or 1-column on mobile.
  6. Autofill-compatible login styling.
    Override browser autofill colors with -webkit-box-shadow: 0 0 0 1000px var(--card) inset and -webkit-text-fill-color: white/85 so the pre-filled email/password don't break the dark theme with white input backgrounds.
  7. Global search or remove it.
    The header search bar currently raises expectations it can't meet. Either implement a Cmd+K command palette that searches products, parcelles, operators, and stock levels, or replace it with page-specific context (breadcrumbs, quick stats).
